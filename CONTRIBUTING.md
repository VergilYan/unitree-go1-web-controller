# Contributing to GO1 Web Controller

Thank you for your interest in contributing to the GO1 Web Controller project! We welcome contributions from developers of all skill levels.

## 📋 How to Contribute

### 1. Fork the Repository

Click the "Fork" button at the top right of the GitHub page to create your own copy of the repository.

### 2. Clone Your Fork

```bash
git clone https://github.com/yourusername/go1-web-controller.git
cd go1-web-controller
```

### 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
# or for bug fixes
git checkout -b bugfix/your-bug-fix
```

### 4. Make Your Changes

- Follow the existing code style
- Add comments where necessary
- Test your changes thoroughly

### 5. Commit Your Changes

```bash
git add .
git commit -m "Add a clear commit message describing your changes"
```

### 6. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### 7. Create a Pull Request

Go to the original repository and click "New Pull Request" to submit your changes for review.

## 📝 Code Style Guidelines

- Use 4 spaces for indentation (not tabs)
- Use descriptive variable names
- Add comments for complex logic
- Follow PEP 8 standards for Python code
- Keep lines under 80 characters where possible
- Use snake_case for Python functions/variables
- Use PascalCase for Python classes

## 🧪 Testing

Before submitting a pull request:

1. Test your changes in SAFE MODE first
2. If modifying robot control code, test with a real robot if possible
3. Ensure all existing functionality still works
4. Check for any errors or warnings

## 📄 Documentation

If you add new features:

1. Update the README.md with information about the new feature
2. Add comments in the code explaining how it works
3. Consider adding documentation in the docs/ folder

## 🐛 Reporting Issues

If you find a bug or have a feature request:

1. Check if the issue already exists in the GitHub Issues
2. If not, create a new issue with:
   - Clear title describing the problem
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
   - Screenshots (if applicable)

## 📞 Getting Help

If you need help with contributing:

- Check the README.md for usage instructions
- Look at existing code for examples
- Open an issue to ask questions
